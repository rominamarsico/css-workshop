### CSS classes

<div>

  <div>

  ✔️Performanter als CSS-in-JS Lösungen

  </div>

  <div>

  ✔️Trennung von Logik und Design

  </div>

  <div>

  ❌ Globaler Scope

  </div>

  <div style={{ marginLeft: "3rem"}}>

  -> Klassennamen sollten nicht mehrfach vergeben werden

  </div>

  <div style={{ marginLeft: "3rem"}}>

  -> Verwendung von css-modules für lokalen Scope

  </div>

  <div>

  ❌ Potentiell uneindeutige Namensgebung

  </div>

  <div>

  ❌ Erfordert viel Disziplin und Erfahrung, ansonsten schwere Wartbarkeit

  </div>

</div>

<Notes>

- Performanter: weil der Browser hier keinen JS Code entpacken/ ausführen muss, um an den CSS Code ranzukommen
- Klassennamen: Styles werden überschrieben bei doppelt genutzten Namen
- uneindeutige Namensgebung: wenn man (wieso auch immer) den 10ten Button styled, braucht man kreative Namen für die Klassen
- Weiterer Negativpunkt für CSS Klassen: Wie werden sie eingesetzt? Siehe Next Slides

### Mobile First

- Inhalte: weniger ist mehr
- Interaktive Elemente hervorheben
- Keine Hover-Effekte möglich
- Keine Fullscreen Fotos
- Auf realen Endgeräten testen

<Notes>

- Inhalte zuerst für mobile Geräte auslegen: weniger ist mehr, kleiner Screen = weniger Platz
- Interaktive (klickbare) Elemente groß darstellen: Finger Klick braucht mehr Platz als Maus-Klick
- Keine Hover-Effekte möglich
- Keine Fullscreen Fotos: Fotos in Hoch- oder Querformat funktionieren nicht bei allen Displaygrößen
- Auf realen Endgeräten testen, nicht auf Simulatoren verlassen

</Notes>
